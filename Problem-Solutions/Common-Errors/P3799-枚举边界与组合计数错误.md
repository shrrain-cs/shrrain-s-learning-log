## 错误现象

1. 循环时边界错误，导致i为偶数时正常，但为奇数时会少一种情况
2. 计算数学公式时，本应是三数相乘，却只乘了前两个数

## 错误原因

### 1. 统计边界条件时出现疏忽

原来写成：

```cpp
for (int j = 1; j < i / 2; j++) {
    // ...
}
```

例如，当 `i = 5` 时，`i / 2 = 2`，循环条件是 `j < 2`，只能枚举到 `j = 1`，会漏掉 `j = 2`。

正确写法：

```cpp
for (int j = 1; j < (i + 1) / 2; j++) {
    // ...
}
```

### 2. 将较长数学式子一次写完易出错 + 没有提前在纸上记录数学公式

原来只写了：

```cpp
ans = (ans % MOD + v[i / 2] * (v[i / 2] - 1) / 2 % MOD) % MOD;
```

实际还需要乘上 `Cvi2`：

```cpp
ans = (ans % MOD + v[i / 2] * (v[i / 2] - 1) / 2 % MOD * Cvi2 % MOD) % MOD;
```

事实上，更推荐拆成中间变量：

```cpp
int same = v[i / 2] * (v[i / 2] - 1) / 2 % MOD;
ans = (ans + same * Cvi2 % MOD) % MOD;
```

## 防错总结

1. 举不同的例子考虑边界情况，确保多个例子万无一失后再确定边界
2. 做数学运算时，根据逻辑关系分式计算，或将数学公式在纸上完整写出后再写到程序上
