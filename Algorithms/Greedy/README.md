# 贪心算法 | Greedy Algorithm

贪心是一种“只看眼前最优”的算法思想——每一步都做出当前看起来最好的选择，并相信这个局部最优最终能导向全局最优。 和高精度、排序这些有固定模板的算法不同，贪心没有统一的代码框架，每道题的策略都可能不一样。做贪心题的核心不在于套模板，而在于**想清楚为什么局部最优就是全局最优**。

## 核心思想

贪心算法的一般步骤：

1. **确定贪心策略**：在当前状态下，什么样的选择是最优的？
2. **排序或维护数据结构**：根据策略对数据排序（如按结束时间、按单位价值），或用优先队列动态维护最优解。
3. **逐个处理**：按顺序做出贪心选择，更新状态。
4. **验证正确性**：问自己：有没有反例能推翻我的策略？

做贪心题的关键不是记住每道题的解法，而是养成一个习惯——**每次凭直觉写出贪心策略后，先想反例**。如果找不到反例，再用“决策包容性”或“最优子结构”去说服自己。

## 学习心得

- **区间调度（P1803）**：这道题让我第一次理解了什么叫“决策包容性”——选结束最早的，绝对不会比其他选择更差，因为它把最多的时间留给了后面的区间。
- **排队接水（P1223）**：这道题教会我“排序不等式”的思想。等待时间的累加本质上是前缀和的累加，让小的数在前面被累加更多次，总和自然最小。
- **部分背包（P2240）**：和0-1背包不同，部分背包可以分割物品，所以贪心成立。只需要排好序后按性价比顺序拿就行。
- **删数问题（P1106）**：这道题卡我最久。核心是理解“删掉第一个峰顶”为什么能让剩下的数最小——因为高位数字对结果的影响远大于低位。
- **合并果子（P1090）**：这道题让我第一次接触优先队列。和前面四道题不同，这道题的数据在合并过程中会动态变化，必须用优先队列而不是一次性排序。这也是哈夫曼树的经典应用。
- **容易出错的地方**：排序规则写错（漏掉次要条件）、忘记处理边界（如结果为0时输出0）、用静态思维解动态问题（如合并果子用固定公式）。
